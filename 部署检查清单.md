# ✅ 部署检查清单

用这个清单确保每一步都完成了！

---

## 📋 第一步：Supabase 配置

- [ ] 1. 在 supabase.com 注册账号
- [ ] 2. 创建新项目 `love-website`
- [ ] 3. 等待项目创建完成（2-3分钟）
- [ ] 4. 复制 Project URL
- [ ] 5. 复制 anon public 密钥
- [ ] 6. 更新本地 `.env.local` 文件
- [ ] 7. 在 SQL Editor 运行建表 SQL
- [ ] 8. 创建 Mao 的账号（邮箱 + 密码）
- [ ] 9. 创建 Pi 的账号（邮箱 + 密码）
- [ ] 10. 记录两个账号的邮箱和密码 ✍️

---

## 📋 第二步：GitHub 推送

- [ ] 1. 打开终端，cd 到项目文件夹
- [ ] 2. 运行 `git init`
- [ ] 3. 运行 `git add .`
- [ ] 4. 运行 `git commit -m "Initial commit"`
- [ ] 5. 在 github.com/new 创建私密仓库
- [ ] 6. 运行 `git remote add origin ...`
- [ ] 7. 运行 `git branch -M main`
- [ ] 8. 运行 `git push -u origin main`
- [ ] 9. 在 GitHub 看到代码已上传 ✅

---

## 📋 第三步：Vercel 部署

- [ ] 1. 访问 vercel.com
- [ ] 2. 用 GitHub 登录
- [ ] 3. 点击 "Add New..." → "Project"
- [ ] 4. 选择 `love-website` 仓库
- [ ] 5. 点击 "Import"
- [ ] 6. 添加环境变量 `NEXT_PUBLIC_SUPABASE_URL`
- [ ] 7. 添加环境变量 `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- [ ] 8. 添加环境变量 `NEXT_PUBLIC_START_DATE`
- [ ] 9. 添加环境变量 `NEXT_PUBLIC_PARTNER1_NAME`
- [ ] 10. 添加环境变量 `NEXT_PUBLIC_PARTNER2_NAME`
- [ ] 11. 点击 "Deploy"
- [ ] 12. 等待部署完成（2-3分钟）
- [ ] 13. 看到 "Congratulations!" 🎉

---

## 📋 第四步：测试

- [ ] 1. 打开 Vercel 给的网址
- [ ] 2. 看到登录页面
- [ ] 3. 用 Mao 的邮箱密码登录成功
- [ ] 4. 看到首页和头像
- [ ] 5. 点击"点点滴滴"测试功能
- [ ] 6. 点击"留言板"发送留言
- [ ] 7. 退出登录测试
- [ ] 8. 用 Pi 的邮箱密码登录测试
- [ ] 9. 记录最终网址 ✍️
- [ ] 10. 添加到手机主屏幕 📱
